\chapter{Analýza existujících nástrojů}
\label{chap:existing_tools}

V~této kapitole jsou analyzovány existující nástroje pro monitorování a~sledování statistik vývoje softwaru. Cílem je identifikovat jejich silné a~slabé stránky a~na základě toho vymezit prostor pro vlastní řešení.

Mnoho velkých open-source projektů, jako je například překladač jazyka Rust, si nevystačí s~obecnými nástroji pro monitorování, které nabízí GitHub. Proto si tvoří vlastní nástroje přizpůsobené jejich specifickým potřebám a~workflow.

\section{Shrnutí analýzy a~identifikace mezer}
\label{sec:existing_tools:summary}

GitHub Insights poskytuje pouze základní statistiky ohledně \PR a~issues. Pro detailní zobrazení je nutné znát sledovaný repozitář a~aktivně využívat pokročilé filtry. Výhodou oproti ostatním nástrojům je přímý přístup k~jednotlivým \PR a~issues přímo z~výsledků filtrování. Curl Dashboard nabízí komplexní vhled do celkové historie projektu, včetně dat, která přímo nesouvisejí s~kódem (distribuce commitů podle časových pásem, počty bezpečnostních zranitelností), a~poskytuje tak širší pohled na projekt. Chybí zde však detailní informace o~konkrétních \PR nebo issues. Rustc PR Tracking nabízí přehledný pohled na stav \PR ve frontě ke schválení, ale zcela opomíjí issues a~celkové metriky projektu.

Žádný z~analyzovaných nástrojů přímo nepomáhá s~operativním vývojem, jedná se spíše o~nástroje pro získávání informací než pro řešení konkrétních problémů. Rovněž chybí řešení, které by propojovalo detailní pohled na \PR a~issues s~historickými daty a~aktivitou nad soubory v~repozitáři. Tato mezera je hlavní motivací pro vytvoření vlastního nástroje.
