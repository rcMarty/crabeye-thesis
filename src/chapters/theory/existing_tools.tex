\section{Shrnutí analýzy a identifikace mezer}

github isnisghts poskytuje pouze základní statistiky ohledně pr a issues a je třeba znát daný repozitář pro získání požadovaných informací. oproti tomu to má výhodu že přesně víme jaké issues nebo prs jsou v tomto stavu a můžou se rovnou zobrazit. u curl dashboardu máme kompetní vhled do celkové historie projektu i s informacemi co nutně nesouvisí s kodem a jeho trackováním ale nabízí to širší globálnější pohled na projekt ovšem chybí detailní informace o pr nebo issues. u rustc je zde přehledný vhled an stav pr ale nic zde není o issues. taktéž ani jeden nástroj přímo nepomáhá s vývojem a jsou spíše nástroji pro získávání informací než pro operativní řešení problémů.
