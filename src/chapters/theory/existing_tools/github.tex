\section{Nativní GitHub statistiky}
\label{sec:github_stats}
github nabízí i zíákladní nativní statistiky které jsou dostupné přímo  u projektu. Tyto statistiky jsou v záložce \textit{Insights} a obsahují počet otevřených / zavřených pr, počet kontributorů a jejich aktivitu a další.

\subsection{Obsah a funkce}
Hlavní bod kde je možno získat obecné statistiky a grafy je záložka insights kde nejdůležitější části zahrnují Pulse, Contributors a commits.
u všech statistik lze získat data za určité období které je nejčastěji den, týden, měsíc nebo rok. U některých statistik z důvodu velkého množství dat je toto období omezené například pouze na maximálně poslední měsíc.
\begin{description}
  \item[Pulse] poskytuje rychlý přehled  o aktivních a uzavřených pr, otevřených a uzavřených issues, nejaktivnějších autorech a základní sumarizaci o aktivitách a přidaných a odebraných řádcích kodu. Poskytuje i seznam uzavřených pr za vybrané časové období. Statistiky jsou k dispozici za poslední den, 3 dny, týden a měsíc.
  \item[Contributions] záložka poskytuje informace o nejčastěji přispívajících uživatelích včetně automatizovaných účtů (botů). zobrazuje počet commitů jak celkových tak od každého uživatele zvlášť. počet commitů je dynamicky agregován podle zvoleného časového období ovšem minimálně 1 týden. časové období ,může být vybráno uživatelem od podleních 2 týdnů po celé časové období existence repozitáře. pokud repozitář obsahuje méně než 10~000 commitů tak tento nástroj zobrazuje i počet změněných řádků pro každého uživatele.
  \item[Commits] záložk obsahuje pouze graf celkového počtu commitů za poslední rok kdy počet commitů je agregován podle týdne.
\end{description}

přehledný způsobu detailnějšího zobrazení pr nebo issues v jednom grafu není ale je možné filtrovt v záložce issues podle různých kritérií za pomocí pokročilých filtrů. Tyto filtry umožnují spojovat podmínky pomocí booleanovských operaci a umožnují zanořování zaa pomocí uzávorkování. U tkového filtru dostneme seznam issues nebo prs s celkovým počtem otevřených a zavřených položek.

Dalšíé nástroj je github projects kde ke každé velké funkcionalitě jsou rozdělena issues do stavů určených vlastníkem jako například TODO nebo IN PROGRESS. Tento nástroj ale neposkytuje žádné statistiky a je spíše nástrojem pro organizaci práce než pro získávání informací o projektu.

\subsection{Zhodnocení}
nativní statistiky githubu poskytují pouze základní přehled o celkovém projektu. jejich hlavní výhodou je že existují nativně a není třeba cokoliv tvořit pro jejich získání.

ovšem statistiky ohledně pr nebo issues jsou těžko získatelné a je třeba znát daný repozitář pro získání požadovaných informací. oproti tomu to má výhodu že přesně víme jaké issues nebo prs jsou v tomto stavu a můžou se rovnou zobrazit.
